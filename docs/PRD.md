# Python 코드카타 Chrome Extension - PRD (Product Requirements Document)

## 1. 제품 개요

### 1.1 제품명
**SPARTA Python 코드카타** Chrome Extension

### 1.2 버전
v2.0.0

### 1.3 목적
SPARTA 부트캠프 학생들의 Python 알고리즘 문제 풀이를 자동으로 GitHub에 업로드하고, 진행 상황을 추적하는 크롬 익스텐션

### 1.4 대상 사용자
- SPARTA 부트캠프 데이터 분석가 과정 수강생
- Python 알고리즘 코드카타 참여자

---

## 2. 문제 정의

### 2.1 현재 상황
- 학생들이 알고리즘 문제를 풀고 수동으로 GitHub에 업로드해야 함
- 진행 상황 추적이 어려움
- 어떤 문제를 풀었는지 기억하기 어려움

### 2.2 해결 방안
- 문제 풀이 후 자동으로 GitHub에 코드 업로드
- 시각적 진행률 대시보드 제공
- 난이도별 문제 목록 및 추천 기능

---

## 3. 기능 요구사항

### 3.1 필수 기능 (MVP)

#### F1: GitHub OAuth 연동
- **F1.1**: GitHub OAuth Device Flow 인증
- **F1.2**: 저장소 선택/생성 UI
- **F1.3**: 연결 상태 표시 (사용자 아바타, 이름)
- **F1.4**: 문제 풀이 코드 자동 Push
- **F1.5**: 로그아웃 기능

#### F2: 문제 목록 관리
- **F2.1**: 441개 문제 목록 표시
  - 기존 프로그래머스 180문제
  - 백준 38문제
  - 기초·입문 223문제 (코딩테스트 입문 100 + 코딩 기초 트레이닝 123)
- **F2.2**: 난이도별 그룹화 (L0~L5, 레벨7, 레벨8)
- **F2.3**: 플랫폼별 탭 분리 (프로그래머스 / 백준 / 기초·입문 / 설정)
- **F2.4**: 완료 상태 체크박스 토글

#### F3: 진행률 추적
- **F3.1**: 전체 진행률 표시 (X/441)
- **F3.2**: 플랫폼별 진행률
  - 프로그래머스: X/180
  - 백준: X/38
  - 기초·입문: X/223
- **F3.3**: 난이도별 진행률

#### F4: 코드 자동 감지
- **F4.1**: 프로그래머스 문제 페이지 코드 추출
- **F4.2**: 백준 제출 페이지 코드 추출
- **F4.3**: 정답 여부 자동 감지

#### F5: 사용자 설정
- **F5.1**: 학생 이름 설정 및 저장
- **F5.2**: GitHub 자동 제출 토글
- **F5.3**: 진행 상황 초기화

### 3.2 부가 기능

#### F6: 다음 문제 추천
- **F6.1**: 미완료 문제 중 난이도순 추천
- **F6.2**: 클릭 시 해당 문제 페이지 열기

#### F7: 접기/펼치기
- **F7.1**: 난이도별 문제 목록 접기/펼치기
- **F7.2**: 기초·입문 카테고리별 접기/펼치기

## 4. 비기능 요구사항

### 4.1 성능
- 팝업 로딩 시간: 1초 이내
- GitHub Push 응답 시간: 3초 이내

### 4.2 보안
- OAuth 토큰은 chrome.storage.local에 저장
- 토큰 로그 출력 금지
- Device Flow 방식으로 client_secret 노출 방지

### 4.3 호환성
- Chrome 브라우저 (Manifest V3)
- 프로그래머스 (school.programmers.co.kr)
- 백준 (www.acmicpc.net)

### 4.4 사용성
- 직관적인 UI (sql_submit 프로젝트 기반)
- 팝업 크기: 420x520px
- 한국어 인터페이스

## 7. UI/UX 명세

### 7.1 sql_submit 프로젝트 기반 디자인

> **핵심 원칙**: 기존 sql_submit 크롬 익스텐션과 **동일한 레이아웃, 컴포넌트 구조, 인터랙션 패턴**을 유지하되, **테마 색상만 Python 스타일로 변경**

#### 변경 사항 요약
| 항목 | sql_submit (SQL) | python_submit (Python) |
|------|------------------|------------------------|
| 주요 색상 | 보라색 (#8b5cf6, #a78bfa) | 녹색 (#22c55e, #4ade80) |
| 배경 그라데이션 | #0f0f1a → #1a1a2e | #0a1628 → #0d1b2a |
| 헤더 배경 | #1e1e2e → #2d2d44 | #1b4332 → #2d6a4f |
| 아이콘 테마 | SQL 데이터베이스 | Python 뱀 |
| 탭 구성 | 단일 플랫폼 | 프로그래머스/백준/기초·입문/설정 |

#### 유지 사항 (sql_submit과 동일)
- **팝업 크기**: 420 x 520px
- **폰트**: Pretendard, -apple-system 폴백
- **레이아웃 구조**: 헤더 → 프로필 → GitHub → 추천 → 진행률 → 탭 → 문제목록
- **난이도 접기/펼치기** 인터랙션
- **체크박스 스타일** 및 애니메이션
- **알림 토스트** 디자인
- **진행률 카드** 레이아웃

## 8. 인증 방식

### 8.1 GitHub OAuth Device Flow
1. 사용자가 "GitHub로 로그인" 클릭
2. Device Code 발급 및 표시
3. 사용자가 GitHub 웹사이트에서 코드 입력
4. Access Token 자동 획득 및 저장
5. 사용자 정보 및 저장소 목록 로드

### 8.2 장점
- client_secret 노출 방지
- 순수 클라이언트 앱에서 사용 가능
- GitHub 공식 지원 방식

## 9. 성공 지표

### 9.1 핵심 지표
- GitHub Push 성공률 > 95%
- OAuth 로그인 완료율 > 90%

### 9.2 사용자 경험 지표
- 문제 풀이 → Push 완료 시간 < 5초
- 오류 발생 시 명확한 에러 메시지 제공

---

## 10. 제약 사항

### 10.1 기술적 제약
- Chrome 브라우저만 지원 (Manifest V3)
- GitHub API Rate Limit: 5000회/시간 (인증 시)

### 10.2 보안 제약
- OAuth 토큰은 사용자 브라우저에만 저장
- 익스텐션은 토큰을 외부로 전송하지 않음 (GitHub API만 호출)
